Verify resumebuildz.tech in Resend

Full step-by-step guide using Namecheap DNS + Supabase Auth.

~20 min work + 5-30 min DNS wait

1Create a Resend account

  1. Go to resend.com and sign up with email or GitHub.
  2. Verify your Resend account email.
  3. You'll land in the Resend dashboard.

2Add your domain in Resend

  1. Sidebar: Domains → Add Domain.
  2. Enter resumebuildz.tech (apex, no www.).
  3. Region: us-east-1 (default, fine globally).
  4. Click Add.

Resend shows a table of DNS records. Keep this tab open. Values look roughly like:

TypeHost / NameValuePriority
MXsendfeedback-smtp.us-east-1.amazonses.com10
TXTsendv=spf1 include:amazonses.com ~all—
TXTresend._domainkeyp=MIGfMA0GCSqGSIb3DQEBAQUAA4GNAD... (long)—

3Open Namecheap DNS

  1. Sign in at namecheap.com.
  2. Top-right: Account → Dashboard.
  3. Find resumebuildz.tech in Domain List → click Manage.
  4. Top tabs: Advanced DNS.

You'll see two sections: Host Records (top) and Mail Settings (bottom).

4Switch Mail Settings to Custom MX

Scroll down to Mail Settings → change the dropdown to Custom MX. Required before Namecheap will accept your MX record.

5Add each Resend record in Host Records

Scroll back up to Host Records. Click Add New Record for each row below.

Namecheap quirk: in the Host field, enter just the subdomain (send, not send.resumebuildz.tech). Namecheap appends the domain automatically.

Record A - MX (bounce routing)

TypeMX Record
Hostsend
Valuefeedback-smtp.us-east-1.amazonses.com
Priority10
TTLAutomatic

Record B - TXT (SPF on send)

TypeTXT Record
Hostsend
Valuev=spf1 include:amazonses.com ~all
TTLAutomatic

Record C - TXT (DKIM)

TypeTXT Record
Hostresend._domainkey
Valuepaste the full p=MIGf... string from Resend
TTLAutomatic
DKIM rules: one continuous line, no line breaks, no surrounding quotes. If Namecheap warns "value too long" it's lying, it auto-chunks internally.

Record D - TXT (DMARC, optional but recommended)

TypeTXT Record
Host_dmarc
Valuev=DMARC1; p=none; rua=mailto:your-personal@gmail.com;
TTLAutomatic

Replace your-personal@gmail.com with any inbox you actually check.

6Save each record

Namecheap saves per-row. Click the green checkmark next to each new record to commit it. There's no global save.

Scan existing records for conflicts

  • An apex SPF on Host @ is fine, leave it alone.
  • An apex MX on Host @ is fine, ours is on send.
  • Any old resend._domainkey TXT → delete the duplicate.

7Verify in Resend

  1. Back in Resend → Domains → click resumebuildz.tech → Verify DNS Records.
  2. Namecheap propagates in 5-15 min typically, sometimes up to an hour.
  3. Rows flip to green individually. When all green → Verified.

Terminal sanity check while waiting

nslookup -type=TXT resend._domainkey.resumebuildz.tech 8.8.8.8
nslookup -type=TXT send.resumebuildz.tech 8.8.8.8
nslookup -type=MX  send.resumebuildz.tech 8.8.8.8

Each should return the values you entered. Empty output = not yet propagated, wait longer.

8Create a Resend API key

  1. Resend sidebar: API Keys → Create API Key.
  2. Name: resumebuildz-prod.
  3. Permission: Sending access (not full access).
  4. Domain: restrict to resumebuildz.tech.
  5. Copy the re_... key.
Shown once. Save the key somewhere safe before closing the dialog.

9Save secrets to Supabase

From your project root (D:\Coding\01-products\ResumeBuildz):

# Link CLI to your project (one-time)
supabase login
supabase link --project-ref kxfakzfjhnbbkdlwkkke

# Secrets for the Edge Function
supabase secrets set RESEND_API_KEY=re_your_key_here
supabase secrets set WELCOME_HOOK_SECRET=$(openssl rand -hex 32)
supabase secrets set WELCOME_FROM="ResumeBuildz <noreply@resumebuildz.tech>"
Save the WELCOME_HOOK_SECRET value somewhere. You need the exact same string in step 11.

If openssl isn't on PATH, use Node instead:

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

10Deploy the welcome Edge Function

supabase functions deploy send-welcome --no-verify-jwt

Expected output: Deployed Function send-welcome on project kxfakzfjhnbbkdlwkkke.

11Wire the Postgres trigger

  1. Open supabase/sql/welcome_email_trigger.sql.
  2. Replace placeholders:
    • <project-ref> → kxfakzfjhnbbkdlwkkke
    • <same-secret-as-function> → the exact WELCOME_HOOK_SECRET from step 9.
  3. Supabase dashboard → SQL Editor → paste the edited file → Run.

Expected: no errors, trigger on_user_email_confirmed exists on auth.users.

12Configure Supabase custom SMTP

The Edge Function covers welcome. The 13 auth/notification templates you pasted (Confirm, Reset, Magic Link, Change Email, Reauth, Invite + 7 notifications) are sent by Supabase itself and need their own SMTP.

  1. Supabase dashboard → Project Settings → Auth → SMTP Settings.
  2. Toggle Enable Custom SMTP.
  3. Fill in:
    Hostsmtp.resend.com
    Port465
    Usernameresend
    Passwordyour Resend API key (reuse step 8 or make a second sending-only key)
    Sender emailnoreply@resumebuildz.tech
    Sender nameResumeBuildz
    Minimum interval60
  4. Click Save changes.
Without this, signup and reset emails still come from Supabase's shared sandbox sender.

13End-to-end test

  1. Open an incognito window → https://resumebuildz.tech/login (or local dev).
  2. Sign up with a real email you control.
  3. You should receive the Confirm signup email from noreply@resumebuildz.tech.
  4. Click the link → email confirmed.
  5. Within seconds: the Welcome email arrives.
  6. Try Reset Password → that template arrives.
  7. Sign in → go to /settings/security → change password → Password changed notification arrives.
If all 7 checks pass, you're done. Every user-facing email now ships from your own domain.
